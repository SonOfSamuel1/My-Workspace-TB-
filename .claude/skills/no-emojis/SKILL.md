# No Emojis — Custom Graphics Only

## Rule

**Never use emoji characters** (Unicode emoji, HTML emoji entities like
`&#128197;`, or emoji shortcodes) in any user-facing UI code. This applies to
buttons, labels, headings, status indicators, badges, tooltips, and any other
visible element.

## What to Use Instead

Use **inline SVG icons** with these conventions:

- **Size**: `width="14" height="14"` for inline/button icons,
  `width="16" height="16"` for standalone icons, `width="20" height="20"` for
  larger UI elements
- **Style**: Stroke-based
  (`fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round"`)
  so icons inherit the parent text color
- **Alignment**: Wrap the SVG in a container with
  `display:inline-flex;align-items:center;gap:4px;` and give the SVG a class
  with `flex-shrink:0;`
- **Accessibility**: Include a descriptive `aria-label` or pair with visible
  text

## When This Applies

- All ActionOS view files (`todoist_views.py`, `home_views.py`, `code_views.py`,
  `dashboard_shell.py`)
- Any new UI component or button added to any app in this workspace
- Status indicators, notification badges, tab labels
- Any HTML generated by Python or TypeScript in this monorepo

## Exceptions

- Log messages and internal debug strings may use text descriptions instead of
  icons (no emoji there either — just text)
- Third-party libraries that render their own emoji are not in scope

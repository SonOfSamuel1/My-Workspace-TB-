# Quick Setup Guide - Google Docs MCP Server

Follow these steps to get the Google Docs MCP server running.

## Step 1: Install Dependencies

```bash
cd servers/google-docs-mcp-server
npm install
```

## Step 2: Get Google Cloud Credentials

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create a new project or select existing
3. Enable APIs:
   - Google Docs API
   - Google Drive API
4. Create OAuth 2.0 Client ID:
   - Navigate to "APIs & Services" → "Credentials"
   - Click "Create Credentials" → "OAuth 2.0 Client ID"
   - Choose "Desktop app" as application type
   - Download the JSON file
5. Save as `credentials.json` in this directory

## Step 3: Build the Server

```bash
npm run build
```

## Step 4: Authorize Access

```bash
node dist/auth.js
```

Follow the prompts:
1. Visit the displayed URL
2. Grant permissions to your Google account
3. Copy the authorization code
4. Paste it into the terminal

This creates `token.json` with your access credentials.

## Step 5: Configure Claude

### For Claude Code

Add to `.mcp.json` in your workspace root:

```json
{
  "mcpServers": {
    "google-docs": {
      "command": "node",
      "args": [
        "/Users/yourusername/path/to/My-Workspace-TB-/servers/google-docs-mcp-server/dist/index.js"
      ]
    }
  }
}
```

### For Claude Desktop

Edit `~/Library/Application Support/Claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "google-docs": {
      "command": "node",
      "args": [
        "/Users/yourusername/path/to/My-Workspace-TB-/servers/google-docs-mcp-server/dist/index.js"
      ]
    }
  }
}
```

Replace `/Users/yourusername/path/to/` with your actual path.

## Step 6: Restart Claude

Restart Claude Code or Claude Desktop for the changes to take effect.

## Verification

Once configured, you should be able to:

```
User: Create a new Google Doc titled "Test Document"
Claude: [Creates the document using the google-docs MCP server]
```

## Common Issues

### Missing credentials.json
**Error:** "Credentials file not found"
**Solution:** Complete Step 2 - download credentials from Google Cloud Console

### Missing token.json
**Error:** "Token file not found"
**Solution:** Run `node dist/auth.js` to authorize

### API not enabled
**Error:** "Google Docs API has not been used..."
**Solution:** Enable the APIs in Google Cloud Console (Step 2)

### Permission denied
**Error:** "Access denied"
**Solution:** Re-run authorization with `node dist/auth.js`

## File Checklist

After setup, you should have:
- ✅ `credentials.json` (from Google Cloud Console)
- ✅ `token.json` (generated by auth.js)
- ✅ `dist/` directory (generated by npm run build)
- ✅ `node_modules/` directory (generated by npm install)

## Security Reminder

**Never commit these files:**
- `credentials.json` - Contains OAuth2 client secrets
- `token.json` - Contains access/refresh tokens
- `.env` - May contain sensitive paths

They are already in `.gitignore`.
